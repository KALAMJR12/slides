Intro to HTML

HTML = HyperText Markup Language

  • Structures content on the web
  • Uses <tags> like <h1>, <p>, <div>
<p>Hello World!</p>
HTML is markup, not programming. Tags come in pairs. Here, <p> defines a paragraph.

HTML Boilerplate

<!DOCTYPE html>
<html>
<head>
  <title>InstiOLX</title>
</head>
<body>
  <h1>Welcome to InstiOLX</h1>
</body>
</html>
Boilerplate is the skeleton of every webpage. Head = backstage, Body = stage.

Header & Navbar

<header>
  <div id="logo">InstiOLX</div>
  <nav>
    <a href="#">Home</a>
    <a href="#">Categories</a>
    <a href="#">Login</a>
    <a href="#">Sell</a>
  </nav>
</header>
The logo is text for now. Each nav link is an <a> tag.

Banner Image

<section class="banner">
  <img src="banner.jpg" alt="welcome banner">
</section>
Every marketplace site has a banner image. The alt helps accessibility.

Search Bar

<div class="search-bar">
  <input type="text" placeholder="Search products..." required>
  <button>Search</button>
</div>
Inputs capture text. Buttons trigger actions (later with JavaScript).

Welcome Text

<div class="welcome">
  <h2>Discover amazing products at unbeatable prices!</h2>
</div>
A simple div with heading text for styling.

Slideshow

<div class="slideshow">
  <div class="slide image1"></div>
  <div class="slide image2"></div>
  <div class="slide image3"></div>
</div>
/* CSS */
.slide {
  position: absolute;
  width: 100%;
  height: 100%;
  background-size: cover;
  opacity: 0;
  animation: fade 9s infinite;
}
Pure CSS slideshow. Each slide is a background image fading in/out.

Product Grid

<div class="product-grid">
  <div class="product-card">
    <img src="product1.jpg" alt="Product 1">
    <h3>Product 1</h3>
    <p>$20</p>
    <button>Add to Cart</button>
  </div>
</div>
/* CSS */
.product-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(220px, 1fr));
  gap: 20px;
}
CSS Grid arranges items in rows/columns that adjust automatically.

Footer

<footer>
  <p>© Developed by Talents Hive</p>
</footer>
Footers usually have links, contact, policies. Keep them simple.
